# 电商运营助手（GMV 诊断系统）

面向跨境电商招商运营场景的本地化 AI 决策助手。

本项目为 TEMU 跨境电商运营工作流设计，用来解决“销售数据散落在 Excel 中、品类经验无法沉淀、季节性需求难以预判”的问题。系统通过本地化部署，把销售数据、产业知识、节庆日历和 AI 检索能力整合在一起，帮助运营人员提前识别备货窗口、沉淀反常识消费规律，并辅助判断后续招商方向。

项目状态：核心功能已完成，正在持续迭代中。当前版本适用于个人运营工作流，真实业务数据仅保存在本地 SQLite 数据库中，不上传到 GitHub。

## 系统定位

这不是一个单纯展示数据的看板，而是一个面向招商运营工作的“会推理的运营助手”。

| 传统数据分析方式 | 电商运营助手 |
| --- | --- |
| 展示过去发生了什么 | 辅助判断为什么发生，以及接下来该做什么 |
| 依赖人工透视表和重复整理 | 上传 CSV / Excel 后自动计算销售高峰月、价格带、Top10 商家 |
| 经验存在个人脑子里 | 将产业知识、反常识规律、节日消费习惯沉淀到本地知识库 |
| 靠人工盯日历备货 | 节日和高峰月自动生成备货提醒 |
| 跨品类规律靠个人感觉 | AI 先查本地知识库，再基于相似类目做辅助推理 |
| 信息散乱 | 原始数据、分析结论、商家资源、运营智慧库四库联动 |

## 当前已实现能力

### 1. 销售数据上传与自动分析

- 支持导入 CSV / Excel 销售数据。
- 数据写入本地 SQLite，关闭页面或重启电脑后仍可保留。
- 支持德国、西班牙、美国等多国家数据，核心记录带有 `country` 字段。
- 系统会自动完成：
  - 按月汇总销售额。
  - 识别高于均值的销售高峰月份。
  - 基于高峰月 Top10 商家的销售额和销量计算平均售价。
  - 生成建议价格带：均价 * 0.95 至 均价 * 1.05。
  - 将分析结果写入分析结论库。
  - 将 Top10 商家写入商家资源库。

### 2. 四语类目树

- 内置英语 / 西班牙语 / 德语 / 中文四语类目树。
- 类目树文件保存在 `知识库/类目树-多语言对照.csv`，便于后续手动维护。
- 系统在分析数据时会尽量把销售数据中的类目名称匹配到标准类目。
- 当前版本以三级类目为主要分析粒度，后续会继续增强一级、二级、三级类目的级联筛选和冲突确认。

### 3. 智能日历与备货提醒

- 首页提供月历视图。
- 蓝色标签表示节日。
- 橙色标签表示商品或类目的备货提醒。
- 节日信息来自节日日历 / 运营智慧库。
- 销售高峰月会按规则倒推生成备货提醒，方便提前准备招商和运营动作。
- 鼠标悬停标签时，可查看对应节日或提醒说明。

### 4. 四类本地知识库

系统把运营信息拆成四类，避免数据和经验混在一起：

| 知识库 | 用途 |
| --- | --- |
| 原始数据库 | 保存上传文件记录和原始销售数据索引 |
| 分析结论库 | 保存销售高峰月、建议价格带、Top10 商家等结构化结论 |
| 商家资源库 | 保存店铺、公司、电话、邮箱、地址、主营类目、备注 |
| 运营智慧库 | 保存节日消费习惯、产业带信息、反常识规律、人工研究结论 |

### 5. 商家资源库

- 每次上传销售数据后，系统自动把高峰月 Top10 商家写入商家资源库。
- 支持手工新增、编辑、删除商家。
- 支持批量导入商家资源。
- 支持按国家、类目筛选，也支持按店铺名 / 公司名模糊搜索。
- 联系电话、邮箱、地址等敏感信息只保存在本地，不进入本仓库。

### 6. 运营智慧库

- 支持手动记录节日、产业链分布、人工深挖结论、反常识消费规律。
- 支持新增、编辑、删除。
- 节日类知识可联动日历蓝色标签。
- 销售规律类知识可被 AI 聊天检索引用。
- 适合沉淀类似“喂鸟器冬季反而热销，因为消费者担心鸟类缺粮”这样的经验。

### 7. AI 聊天助手

- 使用 DeepSeek API 作为大模型能力。
- 聊天前会先检索本地知识库，再把检索结果作为上下文交给模型。
- 本地检索覆盖：
  - 分析结论库
  - 商家资源库
  - 运营智慧库
  - 原始数据库索引
- 涉及具体销售高峰、价格带、商家信息的问题，优先从本地数据库返回。
- 当本地信息不足时，AI 会提示需要补充数据或知识，而不是直接编造结论。
- 回答末尾会标注信息来源，便于判断依据来自哪里。

## 设计中的迭代方向

以下能力是产品设计目标或后续迭代方向，部分能力已经有基础实现，但仍会继续优化：

- 按任意开始月份和结束月份进行跨年分析，而不是只按自然年。
- 分析结论库支持多版本时间范围存储，例如 `2025-06 至 2026-05`。
- 分析结论库、商家资源库增加一级、二级、三级类目级联筛选。
- 上传数据时对同名三级类目但不同一级 / 二级路径的情况进行人工确认。
- 进一步增强 AI 的跨品类推理，例如把“喂鸟器冬季热卖”的规律迁移到相似喂食器产品。
- 从运营智慧库中自动解析“类目 + 热销月份”结构化规律，并生成日历备货提醒。

## 本地运行

### 环境要求

- macOS 或 Windows 均可，本项目当前主要在 macOS 本地使用。
- Python 3.10 或更高版本。
- 依赖见 `requirements.txt`。

### 安装依赖

```bash
pip install -r requirements.txt
```

如果只想最小化运行，核心依赖主要是：

```bash
pip install openpyxl
```

### 配置 DeepSeek

如需启用 AI 聊天，请在项目根目录创建 `deepseek.env` 文件，格式参考 `.env.example`：

```bash
DEEPSEEK_API_KEY=your_api_key_here
DEEPSEEK_API_BASE=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-reasoner
```

真实密钥不要提交到 GitHub。

### 启动服务

双击：

```text
启动GMV诊断系统.command
```

或在终端运行：

```bash
python3 GMV诊断服务.py
```

然后打开：

```text
http://127.0.0.1:8780/
```

## 数据安全

本仓库只保存代码、页面、类目树样例和产品说明，不保存真实业务数据。

以下内容已被 `.gitignore` 排除，不应上传到 GitHub：

- `用户数据/`：本地 SQLite 数据库、备份、导出文件、上传记录。
- `卖家精灵数据表/`：原始销售数据。
- `deepseek.env` / `deepseek.env.rtf` / `.env`：API 密钥。
- 商家联系人、邮箱、地址等敏感资料。
- 任何真实销售明细、商家资料、未脱敏截图。

建议默认使用私有仓库保存代码备份。如果未来要公开展示，请先确认仓库中没有真实业务数据、密钥、商家联系方式和平台敏感信息。

## 项目背景

本项目来自 TEMU 跨境电商招商运营场景。作者日常负责招商和运营工作，需要持续判断不同国家、不同品类、不同季节的消费机会，并把这些判断转化为商家沟通、备货提醒和招商方向。

作者具备 2 年 TEMU 平台运营经验，管理 140+ 商家，月均 GMV 千万级；同时具备 6 年产业投资与行业研究背景。这个项目的目标不是做一个复杂的数据平台，而是把实际运营工作中的经验、数据和行动提醒集中到一个简单易用的本地工具里。
